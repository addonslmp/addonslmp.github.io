# Multiplugin для Lampa

**Multiplugin** — это менеджер плагинов для приложения **Lampa**, который позволяет удобно устанавливать и управлять большим количеством плагинов из одного интерфейса.

Плагин работает как **каталог и установщик расширений**, упрощая поиск и установку плагинов.

После установки плагины работают как обычные расширения Lampa и **не зависят от мультиплагина**.

---

# Установка

1. Откройте **Lampa**
2. Перейдите в:

```
Настройки → Расширения → Добавить плагин
```

3. Вставьте ссылку:

```
https://addonslmp.github.io/multiplugin.js
```

4. Подтвердите установку.

После этого в настройках появится раздел **Мультиплагин**.

---

# Основные функции

Multiplugin предоставляет несколько инструментов для управления плагинами.

## Синхронизация плагинов

Кнопка **Синхронизация плагинов** загружает актуальный список плагинов.

Во время синхронизации:

- загружается JSON со списком плагинов
- создаются категории
- обновляется информация о плагинах

Рекомендуется выполнять синхронизацию:

- после первой установки
- периодически для получения новых плагинов

---

## Установка плагинов

После синхронизации становятся доступны категории плагинов.

Чтобы установить плагин:

1. Откройте категорию
2. Выберите плагин
3. Подтвердите установку

После установки:

- плагин добавляется в **расширения Lampa**
- он начинает работать как обычный плагин

Таким образом мультиплагин выполняет роль **установщика и каталога**.

---

## Установленные плагины

Раздел **Установленные плагины** показывает плагины, которые были установлены через мультиплагин.

В этом разделе можно:

- увидеть список установленных плагинов
- проверить какие плагины были добавлены
- быстро перейти к установленному плагину

В этом разделе можно получить быстрый доступ к плагинам установленным через мультиплагин.

---

## Установить только онлайн

Функция **Установить только онлайн** автоматически устанавливает все плагины из категории **Онлайн**.

После нажатия:

- мультиплагин получает список онлайн-плагинов
- устанавливает их в расширения Lampa
- плагины становятся доступны сразу после установки

Это удобно для пользователей, которым нужен только онлайн-контент.

---

# Как работает Multiplugin

Multiplugin не хранит плагины внутри себя.

Он выполняет три основные задачи:

1. Загружает список плагинов
2. Показывает их пользователю
3. Устанавливает выбранные плагины в систему расширений Lampa

После установки плагины работают **независимо от мультиплагина**.

---

# Хранение списка плагинов

Список плагинов хранится в отдельном JSON файле.

Ссылка:

```
https://addonslmp.github.io/sources/plugins_mp.json
```

Этот файл содержит информацию о всех доступных плагинах.

---

# Структура JSON плагинов

Пример структуры:

```json
    {
      "name": {
        "ru": "МегаПлагин",
        "uk": "МегаПлагін",
        "en": "MegaPlugin"
      },
      "url": "https://megaplugin.com/plugin.js",
      "description": {
        "ru": "Лучший плагин в мире.",
        "uk": "Найкращий плагін у світі.",
        "en": "Best plugin in the world."
      },
      "category": {
        "ru": "Разное",
        "uk": "Різне",
        "en": "Misc"
      }
    }
```

Основные поля:

| Поле | Описание |
|-----|-----|
| name | название плагина |
| url | ссылка на файл плагина |
| description | описание плагина |
| category | категория плагина |

Название, описание и категории плагинов имеют переводы на три языка RU, UK, EU.

# Установка плагина

Когда пользователь устанавливает плагин:

1. Multiplugin получает **URL плагина из JSON**
2. Вызывает API Lampa для установки расширения
3. Плагин добавляется в список расширений

Фактически происходит то же самое, что и при ручной установке через:

```
Lampa → Расширения → Добавить плагин
```

---

# Удаление плагинов

В Multiplugin **можно удалять плагины через раздел «Установленные плагины»** но только те плагины, которые вы установили через сам мультиплагин.

Или удалить стандартным способом Lampa:

```
Настройки → Расширения → удалить плагин
```

После удаления:

- плагин исчезает из системы
- он перестаёт отображаться в списке установленных

---

# Обновление списка плагинов

Список плагинов хранится отдельно от мультиплагина.

Это позволяет:

- добавлять новые плагины
- обновлять ссылки
- исправлять ошибки

без обновления самого мультиплагина.

Поэтому рекомендуется иногда использовать **Синхронизацию плагинов**.
