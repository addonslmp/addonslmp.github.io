**Multiplugin** — сборка подключаемых плагинов для **Lampa**.

Он загружает список плагинов из JSON и позволяет включать или выключать их галочками.
Фактически это менеджер плагинов, который упрощает установку расширений.

---

# Что он делает

* Получает список актуальных плагинов из JSON
* Показывает их по категориям
* Позволяет включать / выключать плагины
* Загружает только выбранные плагины (**lazy-load**)
* Показывает изменения при обновлении списка
* Позволяет экспортировать плагины в обычные расширения

Multiplugin **не содержит плагины внутри себя**.
Он только подключает их по ссылкам.

---

# Установка

1. **Lampa → Настройки**
2. **Расширения**
3. **Добавить плагин**
4. Вставить ссылку:

```
https://addonslmp.github.io/multiplugin.js
```

После установки в настройках появится раздел **«Мультиплагин»**.

---

# Как пользоваться

1. Зайти в раздел **Мультиплагин**

2. Нажать **«Синхронизировать плагины»**

3. Перезагрузить **Lampa**
   (кнопка перезагрузки есть внутри плагина)

4. Поcле этого у вас в меню мультиплагина появятся категории, в которых можно поставить галочки на нужные плагины.

Если захотите удалить какой-либо плагин, то просто уберите галочку и перезагрузите **Lampa**.

---

# Кнопка «Только онлайн»

Включает набор плагинов для онлайн-просмотра.

Это просто быстрый способ включить популярные плагины без ручного выбора.

---

# Экспорт плагинов

Если какие-то плагины понравились, их можно **экспортировать в обычные расширения**.

Есть два варианта экспорта:

1. **Экспорт → Выбрать плагины** - вы можете выбрать плагины в этом пункте меню, можно устанавливать сразу несколько галочек в разных категориях.
2. **Экспорт включённых** - устанавливает те плагины, которые вы включили в самом мультиплагине.

После экспорта:

* они работают как обычные плагины Lampa
* Multiplugin больше ими не управляет
* они останутся даже если удалить Multiplugin
* удаляются только через **Расширения → список плагинов** в настройках Lampa

---

# Если что-то работает не так

Обычно помогает:

```
Выключить все плагины
+
перезагрузка Lampa
```
**Важно понимать, что те плагины которые вы экспортировали в **Расширения Lampa** они не отключаются в мультиплагине. Их нужно отключить либо удалить вручную в расширениях.**


# Источник списка плагинов

Список плагинов хранится здесь:

```
https://addonslmp.github.io/sources/plugins_mp.json
```

Multiplugin загружает его при синхронизации.

---

# Техническая часть

Multiplugin состоит из трёх частей:

```
JSON источник
      ↓
Multiplugin
      ↓
Lampa plugins
```

### JSON

Содержит список плагинов.

### Multiplugin

Отвечает за:

* загрузку списка
* отображение
* управление включёнными плагинами

### Lampa

Физически загружает JS-файлы плагинов.

---

# Формат JSON

Пример записи:

```json
{
  "name": {
    "ru": "Мегаплагин",
    "uk": "Мегаплагін",
    "en": "Megaplugin"
  },
  "url": "https://site/plugin.js",
  "description": {
    "ru": "Описание",
    "uk": "Опис",
    "en": "Description"
  },
  "category": {
    "ru": "Разное",
    "uk": "Різне",
    "en": "Misc"
  }
}
```

Главное поле:

```
url
```

**URL — это уникальный идентификатор плагина.**

---

# Lazy-load загрузка

Multiplugin загружает только включённые плагины.

Используется стандартная функция Lampa:

```javascript
Lampa.Utils.putScriptAsync()
```

По сути multiplugin делает:

```html
<script src="plugin.js"></script>
```

---

# Защита от дубликатов

Чтобы один и тот же плагин не загружался несколько раз используется:

```javascript
const loadedPlugins = new Set()
```

Перед загрузкой выполняется проверка:

```javascript
if (loadedPlugins.has(url)) return;
```

После загрузки:

```javascript
loadedPlugins.add(url)
```

Это важно, потому что один и тот же плагин может находиться в нескольких категориях.

# Storage ключи

Multiplugin использует несколько ключей в `Lampa.Storage`.

| Ключ | Назначение |
|-------|--------|
| `multi_plugins_list` | Список плагинов, полученный из JSON |
| `multi_enabled_plugins` | Список URL включённых плагинов |
| `multi_last_update` | Информация о последнем обновлении |
| `updateCache` | Кеш загрузки плагинов |

**Примечания**:
- Все ключи начинаются с `multi_` - чтобы не пересекаться с другими плагинами.
- `updateCache` - Технический кеш загрузки. Используется чтобы избежать повторной загрузки старых ссылок после синхронизации.
---
# Синхронизация
Кнопка **«Синхронизировать плагины»** выполняет:

1. загрузка JSON
2. сравнение списков
3. определение added / removed
4. обновление Storage

---

# Категории

Категории используются **только для интерфейса**.

Один и тот же плагин может находиться в нескольких категориях.

Физически он загрузится только один раз.

---

# Итог

Multiplugin — это просто:

```
список плагинов (JSON)
+
менеджер включения
+
динамическая загрузка JS
```

Он не изменяет Lampa и не вмешивается в работу самих плагинов.
