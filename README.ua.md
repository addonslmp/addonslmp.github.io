**Multiplugin** — збірка підключуваних плагінів для **Lampa**.

Він завантажує список плагінів із JSON та дозволяє вмикати або вимикати їх за допомогою галочок.  
Фактично це менеджер плагінів, який спрощує встановлення розширень.

---

# Що він робить

* Отримує список актуальних плагінів із JSON
* Показує їх за категоріями
* Дозволяє вмикати / вимикати плагіни
* Завантажує лише вибрані плагіни (**lazy-load**)
* Показує зміни під час оновлення списку
* Дозволяє експортувати плагіни у звичайні розширення

Multiplugin **не містить плагінів усередині себе**.  
Він лише підключає їх за посиланнями.

---

# Встановлення

1. **Lampa → Налаштування**
2. **Розширення**
3. **Додати плагін**
4. Вставити посилання:

```
https://addonslmp.github.io/multiplugin.js
```

Після встановлення в налаштуваннях з’явиться розділ «Мультиплагін».

---

# Як користуватися

1. Зайти у розділ **Мультиплагін**

2. Натиснути **«Синхронізувати плагіни»**

3. Перезавантажити **Lampa**
   (кнопка перезавантаження є всередині плагіна)

4. Після цього в меню мультиплагіна з’являться категорії, де можна поставити галочки на потрібні плагіни.

Якщо захочете видалити якийсь плагін, просто зніміть галочку та перезавантажте **Lampa**.

---

# Кнопка «Тільки онлайн»

Вмикає набір плагінів для онлайн-перегляду.

Це просто швидкий спосіб увімкнути популярні плагіни без ручного вибору.

---

# Експорт плагінів

Якщо якісь плагіни вам сподобались, їх можна **експортувати у звичайні розширення**.

Є два варіанти експорту:

1. **Експорт → Обрати плагіни** — у цьому пункті меню можна вибрати плагіни. Можна встановлювати одразу кілька галочок у різних категоріях.
2. **Експорт увімкнених** — встановлює ті плагіни, які ви увімкнули в самому мультиплагіні.

Після експорту:

* вони працюють як звичайні плагіни Lampa
* Multiplugin більше ними не керує
* вони залишаться навіть якщо видалити Multiplugin
* видаляються тільки через **Розширення → список плагінів** у налаштуваннях Lampa

---

# Якщо щось працює не так

Зазвичай допомагає:

```
Вимкнути всі плагіни
+
перезавантажити Lampa
```

**Важливо розуміти, що ті плагіни, які ви експортували в розділ **Розширення Lampa**, не вимикаються у мультиплагіні. Їх потрібно вимкнути або видалити вручну в розширеннях.**

```

```

# Технічна частина

Multiplugin складається з трьох частин:

```
JSON джерело
      ↓
Multiplugin
      ↓
Lampa plugins
```

### JSON

Містить список плагінів.

### Multiplugin

Відповідає за:

* завантаження списку
* відображення
* керування увімкненими плагінами

### Lampa

Фізично завантажує JS-файли плагінів.

---

# Формат JSON

Приклад запису:

```json
{
  "name": {
    "ru": "Мегаплагин",
    "uk": "Мегаплагін",
    "en": "Megaplugin"
  },
  "url": "https://site/plugin.js",
  "description": {
    "ru": "Описание",
    "uk": "Опис",
    "en": "Description"
  },
  "category": {
    "ru": "Разное",
    "uk": "Різне",
    "en": "Misc"
  }
}
```

Головне поле:

```
url
```

**URL — це унікальний ідентифікатор плагіна.**

---

# Lazy-load завантаження

Multiplugin завантажує лише увімкнені плагіни.

Використовується стандартна функція Lampa:

```javascript
Lampa.Utils.putScriptAsync()
```

По суті multiplugin робить:

```html
<script src="plugin.js"></script>
```

---

# Захист від дублікатів

Щоб один і той самий плагін не завантажувався кілька разів використовується:

```javascript
const loadedPlugins = new Set()
```

Перед завантаженням виконується перевірка:

```javascript
if (loadedPlugins.has(url)) return;
```

Після завантаження:

```javascript
loadedPlugins.add(url)
```

Це важливо, тому що один і той самий плагін може знаходитися у кількох категоріях.

# Storage ключі

Multiplugin використовує кілька ключів у `Lampa.Storage`.

| Ключ                    | Призначення                        |
| ----------------------- | ---------------------------------- |
| `multi_plugins_list`    | Список плагінів, отриманий із JSON |
| `multi_enabled_plugins` | Список URL увімкнених плагінів     |
| `multi_last_update`     | Інформація про останнє оновлення   |
| `updateCache`           | Кеш завантаження плагінів          |

**Примітки:**

* Усі ключі починаються з `multi_`, щоб не перетинатися з іншими плагінами.
* `updateCache` — технічний кеш завантаження. Використовується, щоб уникнути повторного завантаження старих посилань після синхронізації.

---

# Синхронізація

Кнопка **«Синхронізувати плагіни»** виконує:

1. завантаження JSON
2. порівняння списків
3. визначення added / removed
4. оновлення Storage

---

# Категорії

Категорії використовуються **лише для інтерфейсу**.

Один і той самий плагін може знаходитися у кількох категоріях.

Фізично він завантажиться лише один раз.

---

# Підсумок

Multiplugin — це просто:

```
список плагінів (JSON)
+
менеджер увімкнення
+
динамічне завантаження JS
```

Він не змінює Lampa і не втручається у роботу самих плагінів.
