# Multiplugin для Lampa

**Multiplugin** — це менеджер плагінів для застосунку **Lampa**, який дозволяє зручно встановлювати та керувати великою кількістю плагінів з одного інтерфейсу.

Плагін працює як **каталог і встановлювач розширень**, спрощуючи пошук та встановлення плагінів.

Після встановлення плагіни працюють як звичайні розширення Lampa і **не залежать від мультиплагіна**.

---

# Встановлення

1. Відкрийте **Lampa**
2. Перейдіть у:

```
Налаштування → Розширення → Додати плагін
```

3. Вставте посилання:

```
https://addonslmp.github.io/multiplugin.js
```

4. Підтвердіть встановлення.

Після цього в налаштуваннях з'явиться розділ **Мультиплагін**.

---

# Основні функції

Multiplugin надає кілька інструментів для керування плагінами.

## Синхронізація плагінів

Кнопка **Синхронізація плагінів** завантажує актуальний список плагінів.

Під час синхронізації:

- завантажується JSON зі списком плагінів
- створюються категорії
- оновлюється інформація про плагіни

Рекомендується виконувати синхронізацію:

- після першого встановлення
- періодично для отримання нових плагінів

---

## Встановлення плагінів

Після синхронізації стають доступними категорії плагінів.

Щоб встановити плагін:

1. Відкрийте категорію
2. Виберіть плагін
3. Підтвердіть встановлення

Після встановлення:

- плагін додається до **розширень Lampa**
- він починає працювати як звичайний плагін

Таким чином мультиплагін виконує роль **встановлювача та каталогу**.

---

## Встановлені плагіни

Розділ **Встановлені плагіни** показує плагіни, які були встановлені через мультиплагін.

У цьому розділі можна:

- побачити список встановлених плагінів
- перевірити, які плагіни були додані
- швидко перейти до встановленого плагіна

У цьому розділі можна отримати швидкий доступ до плагінів, встановлених через мультиплагін.

---

## Встановити тільки онлайн

Функція **Встановити тільки онлайн** автоматично встановлює всі плагіни з категорії **Онлайн**.

Після натискання:

- мультиплагін отримує список онлайн-плагінів
- встановлює їх у розширення Lampa
- плагіни стають доступними одразу після встановлення

Це зручно для користувачів, яким потрібен лише онлайн-контент.

---

# Як працює Multiplugin

Multiplugin не зберігає плагіни всередині себе.

Він виконує три основні завдання:

1. Завантажує список плагінів
2. Показує їх користувачеві
3. Встановлює вибрані плагіни в систему розширень Lampa

Після встановлення плагіни працюють **незалежно від мультиплагіна**.

---

# Зберігання списку плагінів

Список плагінів зберігається в окремому JSON файлі.

Посилання:

```
https://addonslmp.github.io/sources/plugins_mp.json
```

Цей файл містить інформацію про всі доступні плагіни.

---

# Структура JSON плагінів

Приклад структури:

```json
    {
      "name": {
        "ru": "МегаПлагин",
        "uk": "МегаПлагін",
        "en": "MegaPlugin"
      },
      "url": "https://megaplugin.com/plugin.js",
      "description": {
        "ru": "Лучший плагин в мире.",
        "uk": "Найкращий плагін у світі.",
        "en": "Best plugin in the world."
      },
      "category": {
        "ru": "Разное",
        "uk": "Різне",
        "en": "Misc"
      }
    }
```

Основні поля:

| Поле | Опис |
| :--- | :--- |
| name | назва плагіна |
| url | посилання на файл плагіна |
| description | опис плагіна |
| category | категорія плагіна |

Назва, опис та категорії плагінів мають переклади трьома мовами: RU, UK, EN.

---

# Встановлення плагіна

Коли користувач встановлює плагін:

1. Multiplugin отримує **URL плагіна з JSON**
2. Викликає API Lampa для встановлення розширення
3. Плагін додається до списку розширень

Фактично відбувається те саме, що й при ручному встановленні через:

```
Lampa → Розширення → Додати плагін
```

---

# Видалення плагінів

У Multiplugin **можна видаляти плагіни через розділ «Встановлені плагіни»**, але тільки ті плагіни, які ви встановили через сам мультиплагін.

Або видалити стандартним способом Lampa:

```
Налаштування → Розширення → видалити плагін
```

Після видалення:

- плагін зникає з системи
- він перестає відображатися у списку встановлених

---

# Оновлення списку плагінів

Список плагінів зберігається окремо від мультиплагіна.

Це дозволяє:

- додавати нові плагіни
- оновлювати посилання
- виправляти помилки

без оновлення самого мультиплагіна.

Тому рекомендується іноді використовувати **Синхронізацію плагінів**.

# Переваги Multiplugin

- Швидке встановлення плагінів
- Зручна система категорій
- Встановлення онлайн-плагінів однією кнопкою
- Просте оновлення списку плагінів
- Мінімальне навантаження на систему
- Плагіни працюють незалежно від мультиплагіна
